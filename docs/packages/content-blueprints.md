# Content Blueprints Package

This package defines the stable architecture layer of the repository.

## It contains

- reusable model definitions
- section contracts
- starter recipes
- starter-to-model mapping

## Why it matters

Shared model changes can be propagated predictably across starters. Each framework template consumes the same content architecture while preserving framework-native rendering and interaction.
